# B2B 数据许可协议 v1.0.0

> **状态:** 可签署 | **生效日:** 2026-05-21
> **协议编号模版:** ZWF-B2B-{YYYYMMDD}-{LICENSEE}
> **法律适用:** 香港特别行政区法律 | **仲裁:** HKIAC
> **审计:** Stella 独立审计签章（待签）
> **公开版本:** 本文件为公开标准版本。签署时生成带 `signature_hash` 的不可变副本。

---

## §1 定义与解释

### 1.1 核心定义

| 术语 | 定义 |
|------|------|
| **行为数据** | 指由 ZWISERFIT 门店 IoT 传感器系统采集的、经脱敏处理的会员身体行为事件数据，包括但不限于：入场签到时间、训练类型与时长、身体测量指标趋势、设备使用记录。**行为数据不含**：原始视频流、人脸特征向量、个人身份标识信息（PII）、医疗诊断记录。 |
| **许可方** | ZWISERFIT Foundation（数据受托人），注册于香港，持有行为数据的信托管理权。 |
| **被许可方** | 与本协议签署的数据使用方，为在许可范围内使用行为数据的商业实体。 |
| **ZWF-20 标准** | ZWISERFIT 发布的行为数据资产标准 v1.0，定义了行为数据的采集格式、脱敏规范、隐私边界和质量等级。 |
| **有效许可** | 经许可方、被许可方、数据主体（通过MPC签名）三方确认的数据使用授权。 |
| **PDL** | Programmable Data License v1.0，本协议引用的可编程数据许可框架。 |

### 1.2 解释规则

本协议中：
- "包括"意为"包括但不限于"
- 条款标题仅为方便参考，不影响解释
- 如本协议与 PDL 存在冲突，以本协议为准（本协议为 PDL 的 B2B 执行层）

## §2 许可范围与分级

### 2.1 许可分级

被许可方根据商业需求选择以下许可等级之一：

| 等级 | 数据访问范围 | 使用权限 | 费用结构 |
|:----:|-------------|---------|---------|
| **L2** | 聚合统计数据 + 脱敏样本（n<100） | 企业内部分析。禁止转售。禁止衍生数据产品。 | ¥50-150/人/年 |
| **L3** | L2 + 脱敏个体数据（n≥100） | 衍生数据产品开发。需收益分成。 | ¥150-250/人/年 + 收益分成5-15% |

> 注：L0（公开研究）和 L1（学术研究）不适用本 B2B 协议。L4（独家数据合作方）另行协商。

### 2.2 使用边界

被许可方承诺：
1. **不进行重标识：** 不尝试将脱敏数据与任何个人身份信息关联
2. **不转售原始数据：** 不以任何形式向第三方提供原始行为数据
3. **不超出声明用途：** 仅在许可申请时声明的用途范围内使用数据
4. **数据本地化：** 如涉及中国大陆数据主体的行为数据，数据存储和处理须遵守《数据安全法》第36条本地化要求

### 2.3 禁止用途（绝对红线）

被许可方**不得**将行为数据用于：
- 个人保险核保或保费差异化（反歧视）
- 雇佣决策或信用评估
- 执法监控或国家安全用途（DID除名触发）
- 任何可能导致数据主体被歧视、 stigmatization 或权益受损的用途

## §3 数据质量与交付

### 3.1 ZWF-20 质量等级

所有交付数据标注 ZWF-20 质量等级：

| 质量等级 | 标准 | 适用场景 |
|:------:|------|---------|
| **QA** | 门禁×人脸识别双重验证 + 60+ 次训练记录 | 精算级分析 |
| **QB** | 门禁单验证 + 30+ 次训练记录 | 一般商业分析 |
| **QC** | 门禁单验证 + <30 次训练记录 | 趋势分析 |

### 3.2 交付方式

- 数据通过加密 API 接口交付（符合 ZWF-20 API 规范）
- 每次交付附带数据质量报告和哈希存证
- 被许可方接收数据后 48 小时内可提出质量异议

### 3.3 持续更新

- 数据为持续流式交付（非一次性快照）
- 更新频率：月度（L2）/ 周度（L3）
- 被许可方可随时查询当前数据覆盖范围

## §4 费用与支付

### 4.1 L2 费用结构

```
年费 = 基础授权费 + 人均数据费 × 授权人数

基础授权费：¥20,000/年（含 API 接入 + 质量报告 + 合规支持）
人均数据费：¥50-150/人/年（按数据质量等级浮动）
```

### 4.2 L3 费用结构

```
年费 = L2 费用 + 衍生品收益分成

收益分成：衍生数据产品净收入的 5-15%
最低年分成：¥100,000（如当年衍生品收入低于最低门槛，补足差额）
```

### 4.3 支付条款

- 首年费用于协议签署后 30 日内支付
- 续费于每个许可年度开始前 30 日支付
- 收益分成为季度结算，每季度结束后 45 日内支付

## §5 审计权

### 5.1 被许可方审计权

被许可方有权每年一次（费用自理）委托独立审计机构验证：
- 数据质量是否符合 ZWF-20 标准
- 许可方是否履行了数据受托人义务

### 5.2 许可方审计权

许可方（通过 Stella）有权对被许可方进行合规审计：
- 验证数据使用是否超出许可范围
- 验证收益分成计算是否准确
- 审计频率：每年至少一次，有合理理由时可增加

### 5.3 审计证据链

所有审计结果生成哈希存证，纳入 ZWISERFIT 公开注册表（`REGISTRY.md`），审计方名称脱敏。

## §6 数据安全与泄露通知

### 6.1 安全标准

被许可方须维持不低于 ISO 27001 或等效标准的数据安全管理体系。

### 6.2 泄露通知

被许可方在发现数据泄露后：
1. **24 小时内**书面通知许可方
2. **48 小时内**提供初步影响评估
3. **72 小时内**向受影响的数据主体发出通知（如适用）

### 6.3 泄露责任

因被许可方安全过失导致的数据泄露，被许可方承担：
- 数据主体的直接损失赔偿
- 许可方的合理调查费用
- 监管罚款（如适用）

---

## §7 终止与数据处置

### 7.1 终止条件

本协议可在以下情况下终止：
1. 任何一方提前 60 日书面通知（正常终止）
2. 一方实质性违约，守约方给予 30 日补救期后仍未纠正
3. 被许可方破产或进入清算程序
4. 许可方破产保护触发 §7.3

### 7.2 终止后数据处置

协议终止后，被许可方须在 30 日内：
1. 删除所有原始行为数据及其副本
2. 提供删除证明（经第三方审计）
3. 已开发的衍生数据产品可保留，但须停止使用新数据更新

### 7.3 许可方破产保护

如许可方（ZWISERFIT Foundation）进入破产程序：
1. 行为数据**不作为破产财产**，回归数据信托
2. 所有有效许可自动暂停，待继任受托人确认后恢复
3. 被许可方有权要求将未到期费用按比例退还
