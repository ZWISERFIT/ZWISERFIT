### 2.2 进攻性合规策略

ZWISERFIT 采用"进攻性合规"——不是被动等待监管审查，而是主动将合规架构公开：

- ✅ GitHub 公开全部协议文本和标准文档
- ✅ 哈希注册表公开，任何第三方可独立验证数据完整性
- ✅ 合规自检清单公开，监管方可自助审查
- ✅ 版本历史公开，每次协议变更可追溯
- ✅ 独立审计入口公开，Stella 审计报告可查

## 三、数据脱敏合规承诺

### 3.1 零隐私泄露保证

ZWISERFIT 承诺并接受 Stella 独立验证：

1. **原始视频永不离开门店设备** — 店内处理，30秒内删除
2. **人脸特征向量永不存储** — 匹配即丢弃
3. **PII 永不与行为数据关联** — DID 系统隔离
4. **每次数据授权需会员 MPC 签名** — 技术强制，非信用承诺
5. **所有数据出口有日志** — Tristan 独立监控

### 3.2 违反脱敏铁律的后果

- 立即触发 Stella 独立调查
- 违规 Agent 冻结权限
- 24h 内上报三元决策层
- 受影响数据主体收到通知
- 公开注册表记录违规事件（不脱敏违规方名称）

## 四、开源许可策略

### 4.1 分层许可体系

| 内容类型 | 适用许可 | 说明 |
|---------|---------|------|
| 代码 | Apache 2.0 | 标准开源许可 |
| 数据格式定义 | PDL v1.0 | 行为数据专用许可 |
| 协议文本 | CC BY 4.0 | 可自由引用/改编，需署名 |
| 文档/标准 | CC BY 4.0 | 同上 |
| 数据（脱敏统计） | PDL L0 | 仅聚合统计公开 |
| 品牌/商标 | 保留所有权利 | 未经许可不得使用 ZWISERFIT 商标 |

### 4.2 PDL vs 传统开源许可

传统许可（MIT/Apache/GPL）只能覆盖代码，无法表达"行为数据的伦理和法律边界"。PDL 是补充层：

```
代码许可: Apache 2.0 → 你可以 fork 我们的代码
数据许可: PDL v1.0  → 你的行为数据受 PDL 保护
```
